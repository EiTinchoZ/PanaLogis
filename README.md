## ВҝQuГ© es PanaLogis?

PanaLogis es un **sistema de gestiГіn operativa** para empresas de transporte de carga terrestre en PanamГЎ. Centraliza el control de flota, conductores, clientes, Гіrdenes de servicio, mantenimiento y facturaciГіn en una base de datos relacional normalizada hasta 3FN, con una interfaz web que expone esa lГіgica de forma operativa.

La caracterГӯstica tГ©cnica central del proyecto es que **toda la lГіgica de negocio crГӯtica vive en la base de datos** вҖ” no en el cГіdigo de la aplicaciГіn. Seis triggers y dos stored procedures garantizan integridad transaccional, generaciГіn automГЎtica de facturas y trazabilidad de operaciones, sin importar quГ© interfaz acceda al sistema.

> *"La base de datos es el Гәnico punto de verdad de la operaciГіn."*

## Quick Start вҖ” Local

### Requisitos

- Python 3.10+
- XAMPP con MariaDB activo (puerto 3306)

### Setup

```bash
# 1. Clonar el repositorio
git clone https://github.com/EiTinchoZ/PanaLogis.git
cd PanaLogis

# 2. Instalar dependencias
pip install -r requirements.txt

# 3. Importar la base de datos
mysql -u root -p --default-character-set=utf8mb4 < database/panalogis.sql

# 4. Levantar Flask
python app.py

# 5. Abrir en el navegador
# http://localhost:5000
```

### Variables de entorno (opcional)

Crea un archivo `.env` para sobrescribir los valores por defecto:

```env
PANALOGIS_DB_HOST=127.0.0.1
PANALOGIS_DB_PORT=3306
PANALOGIS_DB_USER=root
PANALOGIS_DB_PASSWORD=
PANALOGIS_DB_NAME=panalogis_db
SECRET_KEY=tu-clave-secreta
GROQ_API_KEY=tu-groq-api-key          # Opcional вҖ” activa el copiloto IA
PANALOGIS_AI_MODEL=llama-3.3-70b-versatile
```

### Ejecutar tests

```bash
python tests/runtime_smoke.py
# Expected: 12/12 tests OK
```

## Deploy en Vercel

PanaLogis estГЎ configurado para desplegarse en Vercel con Supabase como base de datos cloud gratuita. El proyecto mantiene MariaDB/MySQL como base oficial local y usa PostgreSQL solo como adapter de hosting para la demo online.

### 1. Crear la base en Supabase

1. Crear cuenta en [supabase.com](https://supabase.com) вҶ’ plan gratuito
2. Crear un proyecto nuevo y esperar a que la base quede activa
3. Abrir **SQL Editor**
4. Ejecutar completo el archivo `database/supabase_postgres.sql`
5. Ir a **Project Settings вҶ’ Database** y copiar la connection string o los datos de host/port/user/password/database

### 2. Deploy en Vercel

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https://github.com/EiTinchoZ/PanaLogis)

O manualmente:

```bash
# Instalar Vercel CLI
npm i -g vercel

# Desde la carpeta del proyecto
vercel

# Configurar variables de entorno en vercel.com вҶ’ Project вҶ’ Settings вҶ’ Environment Variables:
# PANALOGIS_DB_ENGINE   вҶ’ postgres
# PANALOGIS_DB_URL      вҶ’ connection string de Supabase (recomendado)
# PANALOGIS_DB_SSLMODE  вҶ’ require
# SECRET_KEY            вҶ’ cualquier string largo y aleatorio
#
# Alternativa sin URL:
# PANALOGIS_DB_HOST
# PANALOGIS_DB_PORT
# PANALOGIS_DB_USER
# PANALOGIS_DB_PASSWORD
# PANALOGIS_DB_NAME
# SECRET_KEY            вҶ’ cualquier string largo y aleatorio
```

### 3. Verificar

DespuГ©s del deploy, visitar la URL de Vercel. El sistema carga el dashboard con los datos de prueba incluidos en `database/panalogis.sql`.

## Consultas de ejemplo

```sql
-- Estado actual de la flota
SELECT v.placa, v.marca, v.modelo, tv.descripcion, v.estado, v.kilometraje
FROM VEHICULO v
JOIN TIPO_VEHICULO tv ON tv.id_tipo_vehiculo = v.id_tipo_vehiculo
ORDER BY v.estado, v.placa;

-- Rentabilidad por ruta (mes actual)
CALL sp_rentabilidad_ruta(YEAR(CURDATE()), MONTH(CURDATE()));

-- Conductores disponibles (sin orden activa)
SELECT CONCAT(c.nombre,' ',c.apellido) AS conductor, c.categoria_licencia, c.vences_licencia
FROM CONDUCTOR c
WHERE c.estado = 'ACTIVO'
  AND c.id_conductor NOT IN (
    SELECT id_conductor FROM ORDEN_SERVICIO
    WHERE estado IN ('PENDIENTE','EN_TRANSITO'));

-- Consolidado de facturaciГіn
SELECT
    YEAR(f.fecha_emision) AS anio,
    SUM(CASE WHEN f.estado='PAGADA'    THEN f.total ELSE 0 END) AS cobrado,
    SUM(CASE WHEN f.estado='PENDIENTE' THEN f.total ELSE 0 END) AS por_cobrar
FROM FACTURA f
GROUP BY anio;
```

**Funciona en dos modos:**

## Decisiones tГ©cnicas

**ВҝPor quГ© la lГіgica de negocio en la BD?**

Los triggers y stored procedures garantizan integridad sin importar quГ© interfaz acceda al sistema. Si en el futuro se agrega una app mГіvil, una API REST o una integraciГіn externa, las reglas (no asignar conductor ocupado, generar factura al entregar) siguen aplicГЎndose porque viven en la BD, no en la aplicaciГіn.

**ВҝPor quГ© MariaDB/MySQL y no PostgreSQL?**

El stack fue definido por los requerimientos acadГ©micos (XAMPP + MariaDB). Por eso el archivo oficial del proyecto sigue siendo `database/panalogis.sql` y el flujo local usa MySQL/MariaDB. PostgreSQL aparece solo como adapter de deploy para Supabase, con un script paralelo (`database/supabase_postgres.sql`) que permite publicar la demo gratis sin alterar la rГәbrica acadГ©mica del proyecto.

**ВҝPor quГ© Flask y no FastAPI/Django?**

Flask permite mantener el proyecto ligero y entendible para propГіsitos acadГ©micos. La arquitectura de blueprints escala correctamente y el cГіdigo de cada mГіdulo es directo y sin abstracciones innecesarias.

**ВҝPor quГ© un design system propio?**

El sistema visual INK/AMBER/SLATE fue construido a medida (CSS puro, ~1200 lГӯneas) para evitar dependencias de frameworks como Bootstrap que aumentan el peso y reducen el control sobre la UI. Space Grotesk + Space Mono dan una estГ©tica tГ©cnica consistente con el dominio del sistema.
